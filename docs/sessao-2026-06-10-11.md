## 10/06 — App mobile (Capacitor)

Objetivo: transformar a loja web em app Android publicável na Play, reaproveitando
a UI e o backend. Cliente **e** admin no mesmo app.

### Modo de build mobile
- `apps/web` ganhou o modo `VITE_TARGET=mobile` (`vite build --mode mobile`, lê `.env.mobile`).
- `base: './'` (Capacitor serve de `https://localhost`; caminhos absolutos quebram).
- Endpoint do Appwrite **absoluto** no app (a web usa troca de host por `origin`).
  Lógica isolada e testada em `apps/web/src/lib/endpoint.ts` (`resolveEndpoint`, com Vitest).
- `VITE_TARGET=web` fixado nos `.env` da web pro Rollup eliminar (DCE) o código nativo do bundle web.

### Casca nativa
- `apps/mobile`: Capacitor + projeto Android versionado.
- Plugins: botão voltar → vue-router, status bar, splash, navegador in-app (`@capacitor/*`).
  Os plugins precisam estar declarados em **apps/mobile** (onde roda `cap sync`), senão
  o Android não registra o módulo nativo (bug "plugin not implemented" corrigido).
- **Ícone e splash**: gerados da **logo do site** (recortei o escudo da marca), todas as densidades.
- **Assinatura/release**: keystore própria (gitignored), `signingConfigs.release` lendo
  `keystore.properties`; gera **AAB** (Play) e **APK universal** (sideload, via bundletool).

### Checkout do cartão no app
- O cartão MP não redireciona a aba (não há aba): abre o checkout no **navegador in-app**
  (`@capacitor/browser`) e confirma pelo **polling do `reconcile-orders`** que já existe,
  com o `orderId` em memória. PIX funciona sem mudança.

### Login Google nativo (Custom Tabs + App Links)
- O Google **bloqueia OAuth em WebView**, e o Appwrite só aceita retorno em host conhecido.
- Solução: abre o OAuth no **Chrome (Custom Tabs)**; o Appwrite volta para uma **página-ponte**
  (`apps/web/public/app-oauth.html`) que reabre o app via **App Link verificado**
  (`/.well-known/assetlinks.json` no domínio + `autoVerify` no AndroidManifest), e o app
  cria a sessão. Esquema custom fica de fallback.
- **Importante:** não escondemos o botão Google — implementamos o fluxo de verdade.

**Validado no emulador (Pixel):** build, instalação, abertura, dados do Appwrite,
botão voltar, ícone/splash, e o fluxo Google até o Google (Custom Tab) + deep link.

## 11/06 — Push, domínio e correções

### Notificação push de venda (admin)
- **Motor:** Appwrite Messaging com provider **FCM** `fcm-admin` (Firebase obrigatório no Android).
- Firebase: projeto `projeto-messas`, pacote `com.sitepedagogico.app`. Arquivos em `secrets/`
  (gitignored): `google-services.json` (vai no app) e a service account (configurou o provider).
- **App:** ao logar, registra o token FCM como **push target** no usuário Appwrite
  (`apps/web/src/mobile/push.ts`). Toque na notificação abre `/admin/pedidos`.
- **Funções:** `reconcile-orders` e `mp-webhook` disparam o push **junto do Telegram**, no
  mesmo claim atômico `paid_<orderId>` (sem duplicar). Best-effort (nunca quebra o pedido).
- **Validado:** push entregue de verdade no emulador (notificação nativa + toque navega).

### Broadcast de push (admin → todos) — publicidade
- **Todos os usuários** passam a registrar token (não só admin).
- Tela admin **"Notificações"** (`/admin/notificacoes`): título + mensagem + link opcional.
- Ação `broadcast-push` no `admin-ops` (só admin dispara): varre todos os usuários e envia
  em lotes de 100 via Appwrite Messaging. Toque abre o link informado ou a home.
- **Validado:** 10 notificações diversas enviadas → 10/10 entregues (`deliveredTotal: 1` cada).

### Migração de domínio — `.com.br` → `www.sitepedagogico.com`
- **Causa:** o domínio `sitepedagogico.com.br` **venceu (28/05) e o registro.br suspendeu**
  (NXDOMAIN). Produção migrou para **`www.sitepedagogico.com`** (apex `sitepedagogico.com`
  faz 308→www; canônico = www).
- **O que mudou no código:** `.env.mobile`/`.env.production`, `endpoint.ts` (webOrigin),
  `auth.store.ts` (ponte OAuth + match), AndroidManifest (host do App Link), `index.html`/`og.ts` (SEO).
- **App Link reverificado** para o novo domínio no emulador (`verified`).

### Correção: clientes não conseguiam logar
- Toda **escrita** (login, cadastro, checkout, OAuth) retornava **403 "Invalid Origin"**
  porque o domínio novo não estava registrado como **plataforma Web** no Appwrite
  (leituras passam por CORS refletido; escritas não).
- **Fix (Wesley, no Console):** registrar `www.sitepedagogico.com` (e `sitepedagogico.com`)
  como Web platform. Confirmado: login voltou (HTTP 201) e o OAuth redireciona ao Google.
- **Google Cloud:** nada a mudar — o `redirect_uri` aponta para o Appwrite
  (`appwrite.wsgestao.digital`), que não mudou.

### Correção: imagens quebradas (domínio antigo no banco)
- As URLs de imagem no banco apontavam para o `.com.br` morto.
- `scripts/rewrite-domain-com.mjs --apply` reescreveu **118 produtos** (`coverImageUrl`) +
  `site_config` (logo + 3 banners) para `www.sitepedagogico.com`.

### Correção: download não funcionava no app
- O WebView do Android **ignora** o `<a download>` de blob (`saveBlob`) e o `window.open(_blank)`.
- **Fix** (`apps/web/src/mobile/download.ts`): o PDF é baixado por `fetchProductFile` (JWT) e
  salvo via `@capacitor/filesystem`, depois aberto na **folha nativa** (`@capacitor/share`) —
  Salvar em Arquivos / Imprimir / Drive. Links abrem no navegador in-app.
- **Validado:** baixou um PDF comprado real no emulador e abriu o share sheet.

### Correção: foto do produto sumiu no preview do WhatsApp
- A `og:image` era **WebP**, e o WhatsApp/Facebook **não renderizam WebP** no preview.
- **Fix** (`api/og.ts`): a `og:image` agora usa o preview do Appwrite com `output=jpg`
  (JPEG, bucket público). Só afeta robôs de preview.
- **Atenção:** o WhatsApp faz cache do preview por link — para ver na hora, mandar o link com
  `?x=1` no fim (ele trata como novo) ou usar o Facebook Sharing Debugger ("Scrape Again").

## Referência rápida

### Buildar a web (mobile) + sincronizar
```bash
pnpm --filter mobile run sync        # build:mobile + cap sync android
```

### Rodar no emulador/aparelho (Windows)
O `cap run` chama `gradlew` sem `.bat` no Windows e falha; buildar direto:
```bash
# variáveis (PowerShell):
#   JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
#   ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"
cd apps/mobile/android
.\gradlew.bat assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### Gerar o release (AAB + APK universal)
```bash
pnpm --filter mobile run sync
cd apps/mobile/android
.\gradlew.bat bundleRelease         # -> app/build/outputs/bundle/release/app-release.aab
# APK instalável (sideload) via bundletool:
java -jar bundletool.jar build-apks --bundle=app/build/outputs/bundle/release/app-release.aab \
  --output=app-release.apks --mode=universal \
  --ks=sitepedagogico.keystore --ks-pass=pass:<SENHA> --ks-key-alias=sitepedagogico --key-pass=pass:<SENHA>
```

### Deploy de função do Appwrite (sem CLI login)
```bash
cd scripts && node deploy-one.mjs <nome-da-funcao>   # ex.: reconcile-orders, mp-webhook, admin-ops
```

## Configurações importantes

### Appwrite — plataformas (CORS/escrita)
Registradas no Console (projeto `6a1bc2b1000d09c3f5f1`): `localhost` (app),
`www.sitepedagogico.com`, `sitepedagogico.com`. **Toda troca de domínio exige registrar o
novo host** — senão login/cadastro/compra dão 403 "Invalid Origin".

### Appwrite — Messaging
Provider FCM `fcm-admin` (configurado com a service account do Firebase `projeto-messas`).
Push de venda vai por `users=[adminIds]`; broadcast por lotes de usuários.

### Segredos (pasta `secrets/`, gitignored — NÃO versionar)
- `google-services.json` — config do Firebase (copiar para `apps/mobile/android/app/` para buildar).
- `*firebase*adminsdk*.json` — service account (configurou o provider FCM no Appwrite).
- `apps/mobile/android/sitepedagogico.keystore` + `keystore.properties` — assinatura do app.

### Proxy / domínio
- `vercel.json` faz `/v1/*` → `appwrite.wsgestao.digital` (sem condição de host → funciona em qualquer domínio).
- A **web** usa troca de host por `origin` (funciona em qualquer domínio); o **app** usa o
  endpoint absoluto `https://www.sitepedagogico.com/v1`.
